## Установка плагина G-Player Music Proxy

**Серверный прокси-плагин необходим для функций онлайн-поиска и воспроизведения.**

---

### Установка в один клик (рекомендуется)

> ⚠️ **Перейдите в родительскую папку, содержащую директорию SillyTavern, перед запуском команд**

#### 🪟 Windows (PowerShell)

1. Откройте PowerShell
2. Перейдите в папку, где расположен SillyTavern, например:

   ```powershell
   cd D:\AI    # Если SillyTavern находится в D:\AI\SillyTavern
   ```

3. Выполните команду установки:

   ```powershell
   cd SillyTavern && (Get-Content config.yaml) -replace 'enableServerPlugins: false', 'enableServerPlugins: true' | Set-Content config.yaml && cd plugins && git clone https://github.com/Sanjs333/g-player-proxy.git && cd g-player-proxy && npm install && cd ../.. && echo "Установка завершена! Перезапустите SillyTavern"
   ```

#### 🐧 Linux / macOS

1. Откройте Terminal
2. Перейдите в папку с SillyTavern:

   ```bash
   cd ~    # Если SillyTavern находится в домашней директории
   ```

3. Выполните команду установки:

   ```bash
   cd SillyTavern && sed -i 's/enableServerPlugins: false/enableServerPlugins: true/' config.yaml && cd plugins && git clone https://github.com/Sanjs333/g-player-proxy.git && cd g-player-proxy && npm install && echo "Установка завершена! Перезапустите SillyTavern"
   ```

#### 📱 Termux (Android)

1. Откройте Termux
2. Выполните команду установки:

   ```bash
   cd ~ && cd SillyTavern && sed -i 's/enableServerPlugins: false/enableServerPlugins: true/' config.yaml && cd plugins && git clone https://github.com/Sanjs333/g-player-proxy.git && cd g-player-proxy && npm install && echo "Установка завершена! Перезапустите SillyTavern"
   ```

### Ручная установка

Если установка одной командой не удалась, выполните следующие шаги:

**Шаг 1: Включите серверные плагины**

1. Откройте `config.yaml` в корневой директории SillyTavern
2. Найдите строку `enableServerPlugins: false`
3. Замените `false` на `true`
4. Сохраните файл

**Шаг 2: Найдите папку плагинов**

Найдите папку `plugins` в корневой директории SillyTavern.

**Шаг 3: Загрузите плагин**

**Вариант А: С помощью Git (рекомендуется)**

```bash
cd SillyTavern/plugins
git clone https://github.com/Sanjs333/g-player-proxy.git
cd g-player-proxy
npm install
```

**Вариант Б: Скачать ZIP вручную**

1. Перейдите на [github.com/Sanjs333/g-player-proxy](https://github.com/Sanjs333/g-player-proxy)
2. Нажмите зелёную кнопку `Code` → `Download ZIP`
3. Распакуйте архив, переименуйте папку в `g-player-proxy` и поместите её в `SillyTavern/plugins/`
4. Откройте терминал в директории плагина и выполните `npm install`

**Шаг 4: Перезапустите SillyTavern**

Закройте и запустите SillyTavern заново.

---

### Проверка установки

При запуске в консоли должно появиться следующее сообщение:

```
[G-Player Proxy] ✓ Started
```

---

### Устранение неполадок

| Проблема | Решение |
| --- | --- |
| Сообщение при запуске не появляется | Убедитесь, что в `config.yaml` установлено `enableServerPlugins: true` |
| Команда `git` не найдена | Установите Git: <https://git-scm.com/downloads> |
| Команда `npm` не найдена | Установите Node.js: <https://nodejs.org/> |
| Неверная структура папок | Убедитесь, что путь `SillyTavern/plugins/g-player-proxy/index.js` существует |
| Ошибка sed на macOS | Используйте `sed -i '' 's/...'` (с пустыми кавычками) или установите gnu-sed |

## 🎨 Темы индикатора

### Автосинхронизация с темой интерфейса

При включённой опции «Синхронизация с темой интерфейса» темы, соответствующие правилу именования, будут переключаться автоматически.

**Правило именования:** название темы интерфейса должно быть **префиксом** названия темы/пресета индикатора

| Тип | Пример названия |
| --- | --- |
| Тема интерфейса | PixelMint |
| Пресет индикатора | PixelMint |
| Тема индикатора | PixelMint-Style / PixelMint Theme |

### Быстрая привязка пресет–тема

При выборе пресета или темы расширение автоматически предложит подходящую пару.

**Правило привязки:** Название пресета + суффикс = название темы

Поддерживаемые суффиксы: `-Style`, `Style`, `Theme`

| Название пресета | Подходящие темы |
| --- | --- |
| PixelMint | PixelMint-Style, PixelMint Style, PixelMint Theme и т.д. |

## 🎵 Плеер

### Загрузка плейлиста из книги мира

Плеер умеет автоматически читать плейлисты из **книг мира, прикреплённых к персонажу (не глобальных)**.

**Шаги настройки:**

1. Создайте **активированную (зелёная лампочка)** запись в книге мира
2. Задайте ключ: `@BGM@`
3. Заполните поле «Контент» в следующем формате:

```
title: All Alone With You
artist: EGOIST
audioUrl: https://…….mp3
coverUrl: https://…….gif
lyricsUrl: https://…….lrc
=====
title: Yasashii Suisei
artist: YOASOBI
audioUrl: https://…….mp3
=====
title: Close Your Eyes
artist: Honda Michiyo
audioUrl: https://…….mp3
coverUrl: https://…….jpg
lyricsUrl: https://…….lrc
```

**Примечания по формату:**

- Используйте **двоеточие `:`** с пробелом
- Разделяйте треки символами `=====` (не менее трёх знаков «равно»)
- URL обложки и текста — необязательные поля

### Автодобавление BGM (вывод ИИ)

После импорта книги мира индикатора и её глобального включения BGM-теги в ответах ИИ будут автоматически отображаться как интерактивные пузырьки и добавляться в плейлист.

**Формат вывода ИИ:**

```
[bgm]Название трека-Исполнитель[/bgm]
```

> ⚠️ Формат BGM НЕ должен быть обёрнут в \` или \`\`\`

## Динамические темы

Динамические темы создают атмосферные эффекты. Когда ИИ описывает конкретные сцены, вывод специального тега переключает тему.

**Формат тега:**

```
[Theme: ИмяТемы]
```

**Пример:** Когда ИИ описывает сцену со снегопадом, вывод `[Theme: 飘雪]` включит эффект снега.
Длительность отображения регулируется параметром «Длительность темы» (0 — скрывать сразу).

**Динамические темы потребляют ресурсы и могут замедлять работу. Включайте в соответствии с возможностями вашего устройства.**

## ❓ Частые вопросы

### Установка плагина

**Q: Установка одной командой завершилась ошибкой?**
A: Используйте ручную установку и выполняйте шаги по одному.

**Q: Команда `git` или `npm` не найдена?**
A: Сначала установите [Git](https://git-scm.com/downloads) и [Node.js](https://nodejs.org/).

**Q: После перезапуска нет сообщения о запуске плагина?**
A: Проверьте:

1. `enableServerPlugins: true` в `config.yaml`
2. Правильный путь плагина: `SillyTavern/plugins/g-player-proxy/index.js`
3. Команда `npm install` была выполнена в директории плагина

### Индикатор

**Q: Почему я не могу нажать кнопки внутри темы?**
A: Сначала включите «Зафиксировать положение» в настройках.

**Q: Стиль индикатора сломан?**
A: Проверьте, есть ли подходящая тема для текущего пресета. Попробуйте использовать связанные пары тема–пресет или включите постоянный режим и обновите страницу.

**Q: Автоследование за темой не работает?**
A: Проверьте правило именования — название темы интерфейса должно быть префиксом названия темы индикатора.

**Q: Изображения не отображаются в индикаторе?**
A: Хостинг изображений может быть заблокирован. Попробуйте использовать VPN.

**Q: Тема для персонажа не применяется?**
A: Если вы вручную выбрали тему в глобальном выпадающем списке, она временно переопределяет настройки персонажа.

Решения:

1. Переключитесь на другой чат и вернитесь (автосброс)
2. Снова выберите тему в выпадающем списке

Приоритет: Ручной выбор > Настройки персонажа > Следование за интерфейсом > Глобальные настройки

### Плеер

**Q: Пузырьки BGM не появляются?**
A: Убедитесь, что ИИ выводит формат `[bgm]Трек-Исполнитель[/bgm]` без обёртки в блок кода.

**Q: Тема плеера не отображается в списке?**
A: Темы плеера должны начинаться с `Player` в названии, иначе они не будут распознаны.

**Q: ИИ не выводит BGM или динамические темы?**
A: Проверьте, **включена ли книга мира индикатора глобально**, или **измените позицию записи**. BGM-записи можно поставить на **Depth 0**, динамические темы на **@⚙, Depth 0**, или **измените текст записи**.

**Q: BGM не воспроизводится в плеере?**
A: Попробуйте **перезапустить плеер** или **обновить страницу SillyTavern**. Если не помогает — трек может быть недоступен.

**Q: Музыкальный пузырёк не управляет воспроизведением?**
A: Пузырьки используют нечёткое совпадение для определения текущего трека. Если управление не работает, название/исполнитель в пузырьке сильно отличаются от того, что отображает плеер.

**Q: Почему в моём плеере нет функции поиска?**
A: Используйте страницу «Инструменты» для восстановления встроенных элементов и обновления плеера. Если вы изменяли встроенные темы и хотите их сохранить, сначала экспортируйте резервную копию.
